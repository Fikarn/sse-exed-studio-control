# Code of Conduct

## Expected Behavior

This project is a professional engineering space for a local studio control application. Contributors are expected to:

- communicate respectfully and directly;
- keep technical disagreements focused on evidence, tradeoffs, and operator safety;
- respect the architecture boundaries, release posture, and hardware constraints documented in the repository;
- avoid harassment, personal attacks, discriminatory language, or disruptive behavior;
- handle security, safety, and operator-impact concerns responsibly.

## Unacceptable Behavior

Unacceptable behavior includes:

- harassment, intimidation, threats, or sustained disruption;
- discriminatory or demeaning comments;
- publishing private information without permission;
- opening public reports for security vulnerabilities;
- knowingly bypassing validation, release, or safety requirements while presenting work as complete.

## Reporting

Report conduct concerns privately to the repository owner. Report security vulnerabilities through GitHub private vulnerability reporting as described in [SECURITY.md](SECURITY.md).

The maintainer may remove comments, close issues or pull requests, restrict repository access, or take other reasonable steps to preserve a focused and safe engineering environment.
