Lighting workspace — ground-up rethink

The lighting surface is the last legacy-shaped workspace. Today it's three disconnected panels: a 1,130-LOC sidebar that owns fixtures, scenes, groups and settings; a scrolling content panel that still violates the 2560×1440 no-scroll rule; and a DMX monitor that renders raw hex outside ConsoleTheme. Scenes were verified to exist but there is no cue list, no fade timing, no blackout affordance, and no spatial awareness — three real design primitives (cue, fade, beam angle) are not expressed in the UI at all.

Three directions are explored below, each optimised for a different daily job. Direction A — Cue stack wins because scene recall during a live show is the operator's highest-frequency action, and lighting desks world-wide already express that mental model the same way. B — Control grid optimises for commissioning (rare). C — Spatial home optimises for plot-day work (also rare). The recommendation block after the findings matrix argues the call explicitly.

◆ Recommended direction

Ship Direction A — Cue stack. Spatial (C) ships as its secondary tab.

Why A

Frequency wins. During a show, the operator recalls 15–40 cues per hour and edits zero. Commissioning (B's optimum) happens twice per production; plot day (C's optimum) happens once. The workspace should be shaped by the shift that dominates calendar time, and that is the live shift.

Matches existing mental models. Every lighting desk the operator has used — ETC Eos, grandMA, Chamsys MagicQ, even basic Chauvet ShowXpress — presents cues as a vertical stack with a GO bar. Zero re-learning cost. B and C are both clever but novel.

Smallest net-new QML surface. A reuses ConsoleButton, ConsoleSurface, ConsoleStatCard, ConsoleSafetyHoldButton and adds one new primitive (ConsoleCueRow). B needs a grid-cell primitive. C needs a full stage-plot canvas with hit-testing.

What A buys us

  • One glance to "what's live" — the active cue is always at the same visual anchor.
  • Fade as a first-class concept — timing is modelled, not hoped for.
  • GO bar + keyboard — Space = GO, B = blackout, . = snap to live.
  • Spatial isn't lost — it's tab 2, one keypress away. You get both mental models without paying for the switch.
  • Commissioning still works — fixture column supports edit-in-place and a "Patch" modal covers the rarer cases.

What A costs us

  • Commissioning is slightly worse than B. Editing 16 fixtures' DMX addresses in a row is faster in a grid. Acceptable — commissioning is a two-day job twice per production.
  • Blind cue-building is harder than C. You can't see the stage while writing cues. Mitigated by tab-2 spatial preview and a "Preview" toggle.
  • One new primitive required. ConsoleCueRow is a new thing in the library; review burden before Phase C.

Lighting-new — proposed additions to ConsoleTheme

If Direction A ships, these are the additive tokens that land in native/qt-shell/qml/ConsoleTheme.qml. All additive — nothing replaces existing tokens. Seven CCT tokens consolidate the ad-hoc ramp at LightingSidebarPanel.qml:50–72. Direction-B/C-only tokens (fixture-cell*, plot-bg) are listed but marked conditional.

cueBarLive / cueBarNext / cueFadeTint
Left-bar indicator for live / next cues, plus a tint for the fade-in-progress row. All three reuse accentPrimary at different alphas — no new brand colour. Shared by ConsoleCueRow.

readonly property color cueBarNext: Qt.rgba(...accentPrimary, 0.5)
readonly property color cueFadeTint: Qt.rgba(...accentPrimary, 0.14)
cct2700 … cct6500
Seven-stop ramp from 2700K (tungsten) to 6500K (overcast daylight). Replaces the three-stop ad-hoc gradient in LightingSidebarPanel.qml. Used by fixture swatches, CCT sliders, beam-preview tints, and DMX-monitor cool/warm channel colouring.

beamSoft / beamStrong
Two alpha-stops used in beam previews (Direction A hero right column) and spatial plot beam cones (Direction C). Both derived from the warm end of the CCT ramp for consistency. 
readonly property color beamSoft: Qt.rgba(1, 0.92, 0.71, 0.10)

fixtureCell / fixtureCellSel · conditional
Only lands if Direction B ships. Grid-cell surface variants. If A is chosen (recommended), these do not need to be added — the fixture list uses surfaceDefault rows.
plotBg / plotFixtureHalo · conditional
Only lands if Direction C ships. Darker-than-shell canvas so beams read at their intended alpha, plus a selected-fixture halo. If A is chosen, we add these later (Phase D) when the spatial tab gets a second pass.
