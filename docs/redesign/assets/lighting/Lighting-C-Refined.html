Lighting · Direction C refined
Spatial desk

The stage plot stays the anchor — but it no longer eats the entire canvas. A 380 px cue rail on the left and a 440 px persistent inspector on the right give cues and parameters dedicated homes; a 140 px control strip at the bottom hosts group pads, scene recall, and a DMX monitor peek. The plot shrinks by ~37%, still dwarfs every other element, and now every lighting-desk function has a place it lives — not behind a keyboard shortcut, not in a popover that covers what you were trying to see.

This is the pass that proves Spatial-as-anchor can actually ship: continuous editing, always-visible cues, groups and scenes on the surface, DMX reachable without a drawer. It's a single-screen operator surface built like a lighting console, not a diagram viewer.

01 What changed from the first Direction C
Before
Plot at 91% of canvas, cue rail at 300 px
Cue rail was too narrow to host a proper GO bar and legible cue rows; plot-day feel was preserved at the cost of live-show feel.
Now
Plot at 68% of width, cue rail at 380 px
GO bar grows to 44 px tall; cue rows are padded; the rail reads as a first-class panel, not a strip. Plot still anchors the eye.
Before
Edits happened in floating popovers
Opening the intensity/CCT popover covered ~30% of the plot. Pressing Esc or clicking away lost your edit context; continuous editing was painful.
Now
Persistent 440 px inspector on the right
Selection flows into the inspector; it stays. Intensity, CCT, groups, patch, cue membership, all always visible. When nothing is selected, it previews the next cue.
Before
Groups / scenes / DMX behind shortcuts or drawers
The workspace felt like a diagram, not a desk. Getting to a saved "Keynote" scene was two keystrokes and a memorized name; group recall was likewise buried.
Now
140 px control strip along the bottom
Group pads (4 pads, live intensity + CCT chip) · scene recall (5 thumbnails with mini-plots) · DMX peek (88 channels at a glance + expand). Every lighting-desk function surfaces without a click.
Before
Multi-select wasn't a first-class action
You could click fixtures one at a time. Editing "all bars" meant clicking each bar fixture and re-opening popovers.
Now
Drag-lasso selection · inspector gains group mode
Drag across fixtures to select; inspector switches to group mode, shows averaged intensity/CCT with a warning if values are mixed, sliders apply to all.

06 · Tokens & new Console* primitives

--cue-bar-live · --cue-bar-next
Tinted row background + left-border colors for the cue rail. Live is the accent-primary @10%; next is @4%. Both respond to focus/hover with a 1-step lift.
--cue-bar-live: rgba(153,186,146,0.10) --cue-bar-next: rgba(153,186,146,0.04) --cue-fade-tint: rgba(245,158,11,0.22)
New
CCT ramp (7 stops)
Perceptual warm→cool ramp used by the persistent inspector and every group pad's CCT chip. Values sampled from Planckian locus; not a pure hue sweep.
--cct-2700: #ffad4a ■ --cct-3200: #ffc278 ■ --cct-3800: #ffd4a0 ■ --cct-4400: #ffe2bd ■ --cct-5000: #f4eedd ■ --cct-5600: #e6ecfa ■ --cct-6500: #d6e4ff ■
New
--plot-bg · --inspector-bg · --inspector-border
Surface tokens for the stage canvas (darkest) and the inspector (slightly raised). Must be additive to ConsoleTheme, not replace existing surface tokens.
--plot-bg: #050509 --inspector-bg: #0d0e14 --inspector-border: #1e1f29 --plot-fixture-halo: rgba(153,186,146,0.45)
New
Beam gradient tokens
Radial gradients used by SVG beam cones. Warm/cool/sage are three presets; the actual opacity is driven by live fixture intensity (0–100%).
--beam-warm: #ffd38b @ 0.55→0 --beam-cool: #eaf0ff @ 0.32→0 --beam-sage: #99BA92 @ 0.28→0 --beam-soft: alpha 0.22 --beam-strong: alpha 0.55
New
ConsoleStagePlot (new)
SVG canvas wrapper. Takes fixtures (array), viewMode (plan|section|elevation), selectedIds, emits fixtureClicked(id) + lassoSelected(ids). Renders beams via SVG radialGradients (no Qt Quick Effects — parity safe).
property var fixtures: [] property string viewMode: "plan" property var selectedIds: [] signal fixtureClicked(id) signal lassoSelected(ids)
New primitive
ConsoleInspector (new)
Persistent right-rail panel. Four tabs (Fixture, Group, Cue, Patch) auto-switch based on selection: single → Fixture, multi → Group, no selection → Cue preview. Patch is only active in commissioning mode.
property var selection: null property var cueAhead: null property bool commissioning: false signal paramChanged(key, value) signal patchApplied(fx, addr)
New primitive
ConsoleGroupPad (new)
Control-strip tile. Shows group name, fixture count, averaged intensity bar, averaged CCT chip. Mixed values render with "Mixed" suffix. Click → select all; right-click → open group editor.
property string name property int fixtureCount property real avgIntensity property color avgCct property bool mixed: false signal selected()
New primitive
ConsoleSceneCard (new)
Scene recall tile with a 200×56 mini-plot thumbnail auto-generated from the scene's beam state. Click → GOTO; shift-click → open editor. Active scene gets a LIVE badge.
property string sceneName property string cueRef property int cctKelvin property bool isLive: false signal recall() signal edit()
New primitive
ConsoleDmxPeek (new)
44-column ribbon of DMX channel levels. Active = sage, stuck = amber, off = dim-grey. Click "Expand" or press Ctrl+M to open the full 88-channel drawer. Replaces the legacy raw-hex monitor entirely.
property var channels: [] property int frameRate: 44 property bool isLinked: true signal expand()
Replaces raw-hex monitor
ConsoleCueRow (reused from A)
Rail row with live/next/done/end states. Same primitive proposed in the original Direction A spec — we keep it here so either direction can ship the same cue-row component.
property string number property string title property string state // done|live|next|queued|end property real fadeSec: 0
Shared primitive
ConsoleParamSlider (new)
Inspector-specific variant of the existing ConsoleSlider. Adds tick marks every 5%, a 28 px height for dense stacking, and sage fill that tracks the value. Keyboard: ↑↓ = 1%, Shift+↑↓ = 10%, 0–9 = snap.
property real value: 0 property bool showMarks: true property string unit: "%" signal valueCommitted(v)
Extends ConsoleSlider
ConsoleSafetyHoldButton · blackout variant
Existing SafetyHoldButton component gains a "blackout" color variant bound to B. Toolbar + rail foot both host instances; activation requires 400 ms hold to avoid mid-show misclicks.
property string variant: "blackout" property int holdMs: 400 signal held()
Extends existing

07 · If we ship this · Phase C plan

Four commits stacked on redesign/v2.2, mirroring the Planning Phase C shape. Each commit lands clean, is independently reviewable, and the final one rebaselines parity for the eight scenes demonstrated above.

The additive-only constraint holds throughout: engine side only adds fields; QML extends ConsoleTheme and the Console* library, never replaces. All new primitives avoid layer.enabled / MultiEffect so offscreen capture stays green; TZ=UTC is already baked into the parity lane from Planning. Windows and macOS both block.

Commit 1 · Engine
Lighting schema v1 · cues, scenes, groups, stage positions
Rust additive — fixtures gain stage_x / stage_y / rig_z; new Cue record with ordered fixture-state map; new Scene record referencing a cue with a name; Group record grouping fixture IDs. Legacy import (legacy_import.rs) gets defaults for new columns — not touched further. IPC protocol additions only; nothing removed.
Commit 2 · Tokens + primitives
ConsoleTheme additions · 7 new Console* files
New tokens: cue-bar-live/next, 7 CCT stops, 2 surface colors, 3 beam presets. New primitives: ConsoleStagePlot, ConsoleInspector, ConsoleGroupPad, ConsoleSceneCard, ConsoleDmxPeek, ConsoleCueRow, ConsoleParamSlider. C++ adapter gains Q_INVOKABLE for recallScene(id), applyPatch(fx, addr), goNextCue().
Commit 3 · Workspace rewrite
LightingWorkspacePanel · retire Sidebar + Content
New LightingWorkspacePanel.qml with the 380/1740/440 + 140 grid. Delete LightingSidebarPanel.qml (1,130 LOC), LightingContentPanel.qml, and the popover DMX monitor. Route shortcut layer: Space = GO, B = Blackout (held 400 ms), Ctrl+M = DMX expand, G = focus group, S = focus scene, Esc = clear selection.
Commit 4 · Parity rebaseline
8 scenes · 2 consecutive bit-identical runs
Capture the eight scenes rendered above: hero, cue-recall, multi-select, section-view, patch-mode, dmx-unreachable, empty-first-open, fallback-1920. Gate both macOS and Windows CI lanes before the workspace PR opens. Subject: parity: rebaseline lighting for v2.2 Cr.

After Commit 4 the workspace is ready to merge into main — but we wait. Audio is still to come; the merge point for redesign/v2.2 is after every workspace is done. This Phase C lands as a stacked commit series on the redesign branch; v2.2.0 tag cuts only once Dashboard + Setup + Planning + Lighting + Audio are all reshaped.
