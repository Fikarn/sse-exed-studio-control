SSE ExEd Studio Control · Planning workspace redesign · 2560×1440

Four directions for the
Planning workspace.

Planning today reads like a workspace that's embarrassed to be a workspace. The overview strip stacks three decorative labels, the right rail duplicates stat numbers already live in the dashboard header, and product copy calls it an "always-on sidecar" that is in fact a full-takeover tab. These directions pick a lane. A · Board-forward commits to the workspace shape and lets kanban own the viewport. B · True sidecar honors the copy — a 420 px persistent right-edge dock, collapsible to 48 px. C · Focus + Board splits the surface in two: live signals on landing, triage board behind a tab. D · Run-of-show timeline asks whether a kanban is even the right primitive when the operator's first task is "scan run-of-show at a glance."

Direction A · Board-forward workspace

Commit to the workspace. Drop the sidecar copy. Give kanban the viewport.

Direction B · True sidecar

Honor the copy literally. A 420 px dock that coexists with Lighting + Audio.

"An always-on sidecar, contained to its own panel."

The posture this workspace has always claimed. Planning becomes a persistent right-edge dock that never hides the active primary workspace. Three states: collapsed 48 px rail (chips only, running-timer badge), standard 420 px dock (project list, running timers, next-up tasks, New Project), expanded 40 % width overlay for kanban + report. The Dashboard "Projects" tab becomes a dock-toggle chord (⇧P). Biggest win: zero workspace takeover. Biggest cost: kanban lives in a narrower column or behind Expand — the board isn't the landing surface.

Direction C · Focus + Board

Split the surface. Live signals on landing, triage board behind a tab.

"What you need at landing ≠ what the board shows."

The hypothesis: the operator's first glance wants answers — what's running? what's blocked? what's next up? — not a four-column planning abstraction. Landing is a Focus surface with a hero running-timer readout, blocked-handoffs list, next-up-per-project, and a quick-add field. Tabs 1 Focus · 2 Board · 3 Report sit on the shell header; ⇧B jumps to the kanban for deep triage. Focus has a sidecar's density without leaving the workspace paradigm.

Direction D · Run-of-show timeline Proposed 4th

Planning is a schedule, not a kanban. Lanes per project, time left-to-right, clock always visible.

"Scan run-of-show at a glance" is literally task #1. A kanban can't do that — a timeline can.

A timeline pins task #1 from OPERATIONS.md to the UI as its literal layout. X-axis is show time (today 09:00 → 22:00); lanes are projects; tasks are blocks with duration. A vertical "NOW" playhead bisects the view and scrolls time with a keyboard nudge ([/]). Status (todo/running/blocked/done) is block fill; running-timer tasks pulse. Kanban isn't killed — it's a mode toggle (⇧B) for deep triage. Biggest bet: whether operators actually think in time vs. status for this product.

ConsoleTheme deltas · proposed

Tokens and Console* variants this redesign would add. Additive only; nothing breaks.

Keeping with dark-only, sage-as-active, mono-for-data. Each addition is tied to a specific direction so we can discard cleanly if we don't pick that direction.

--surface-dock
#101118
Sidecar dock base (Dir B). Sits one step below surfaceDefault so the dock reads as docked-to-shell, not another card. Drop if B is not chosen.
--rail-collapsed
#0c0d13 · 48 px
48 px collapsed rail variant (Dir B). Darker than surfaceDefault so the collapsed rail reads as chrome, not a surface. Pairs with --surface-dock.
--focus-hero
#0e1215
Focus surface background (Dir C). Cooler undertone than studio900 so it reads as "above the board." Used for the Focus-tab landing hero.
--timeline-track
#12131a · = surfaceDefault
Alias — reuse surfaceDefault for the timeline lane background. Named because Dir D needs to refer to it semantically (Qml: Rectangle { color: theme.timelineTrack }).
--timeline-now
rgba(sage, 0.14)
The "now" playhead tint for Dir D — a 4 px sage vertical line + faint 60 px soft band. Single token so the cross-workspace "current time" marker can be reused (Dashboard timeline, if we ever build one).
ConsoleStatChipRow
variant
Toolbar-height chip row (Dir A, D; also Header Redesign). Four stat chips with mono values inline. Replaces PlanningSummaryGrid everywhere; also deletes the 2-row dashboard stat grid over time. 
property var stats: [{ key, label, value, tone }]
height: toolbarHeight // 44
ConsoleKanbanCard
variant
Card with title, tag row, timer chip, handoff indicator, status left-bar. Used by A, B-expanded, C-board. Accepts a running/blocked variant that paints the left bar.

property bool blocked

ConsoleDock
variant (Dir B only)
Right-edge persistent dock with three sizes (collapsed / standard / expanded). Animates between sizes; host workspace shrinks its own max-width to fit. If we don't pick B, this is the component that should not ship.

ConsoleFocusSurface
variant (Dir C only)
Focus-tab landing: hero running-timer readout, blocked list, next-up list, quick-add field. Pure composition of existing primitives (StatCard, Surface, TextField) — the variant is the layout, not new atoms.
ConsoleTimeline
variant (Dir D only)
Lane-based timeline (Dir D). Header scale row + n project lanes + now-playhead. Heaviest new component; only ships if D is chosen. Accessibility: ←/→ scrub, [/] nudge 15 min, 0 snap to now.
